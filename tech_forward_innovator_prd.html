# Caudate Tech — Product Requirements (PRD) ## Product Overview **The pitch:** Caudate Tech is a consulting and digital marketing partner for enterprises and high-growth teams. The site positions the firm around **engineered growth**: SAP and cloud landscapes, paid media (Google, Meta), analytics, and measurable ROI—delivered with a polished, trustworthy visual system (not a dark “cyber” aesthetic). **For:** Organizations that need serious technical and growth execution, clear service paths, and low-friction contact. **Primary surface:** Responsive web; desktop-first layouts with mobile navigation patterns. **Design direction:** Light, high-clarity UI inspired by Material-style tokens: soft blue-tinted surfaces, **teal primary** (`#006a64`), strong typography, generous whitespace. Professional and architectural—aligned with “enterprise + growth” positioning. **Reference implementations in repo:** [`stitch-tech-site/`](stitch-tech-site/) (Next.js app). --- ## Screens (as implemented) | Route | Purpose | |--------|--------| | **Home** (`/`) | Hero with value prop + **ContactInquiryForm** (hero variant); **“Tools & ecosystems we operate in”** band with monochrome brand marks (Simple Icons + Devicon); stats/value sections; CTAs to contact. | | **Services** (`/services`, nested service pages) | Category navigation and detail pages (SAP, digital marketing, paid media, etc.). | | **Case Studies** (`/case-studies`) | Portfolio / proof points. | | **About** (`/about`) | Mission, values bento, timeline, **team grid (3)** with local headshots. | | **Contact** (`/contact`) | Inquiry form + office/contact details; optional **WhatsApp** link when `NEXT_PUBLIC_WHATSAPP_NUMBER` is set. | | **FAQ** (`/faq`) | Accordion FAQ from [`stitch-tech-site/src/data/faq.ts`](stitch-tech-site/src/data/faq.ts). | | **Privacy Policy** (`/privacy-policy`) | Legal. | --- ## Key flows **Lead capture** 1. User lands on **Home** → reads hero → submits **ContactInquiryForm** or follows link to full **Contact**. 2. User may use **WhatsApp** (floating button and/or form/contact copy) when configured. 3. Form posts to **`/api/contact`** (Nodemailer); newsletter/API routes exist as implemented. **Discovery** 1. User opens **Services** from nav or home → drills into specific service URLs. 2. User opens **FAQ** from navbar or footer. 3. User reviews **About** for leadership: **Ajay Kanwar** (CEO & Founder), **Suraj Kanwar** (CTO & Data Analyst), **Raman Verma** (Chief Marketing Officer); images served from **`/team/*.png`** via `next/image`. --- ## Global UX / chrome - **Navbar:** Brand + primary nav; **two mega-menu triggers** (SAP-focused group + Digital Marketing group) driven by [`stitch-tech-site/src/data/navigation.ts`](stitch-tech-site/src/data/navigation.ts); standalone links include **FAQ** and **Contact** (as wired). - **Footer:** Includes FAQ and other key links. - **WhatsApp:** [`WhatsAppChatButton`](stitch-tech-site/src/components/WhatsAppChatButton.tsx) in root layout when URL can be built from env. ---
Design system (production tokens) ## Color palette (see [`stitch-tech-site/src/app/globals.css`](stitch-tech-site/src/app/globals.css)) - **Primary:** `#006a64` — buttons, links, key accents, ecosystem logo monochrome tint. - **Primary container:** `#16b2a8` — gradients / emphasis. - **Background / surface:** `#f7f9ff` — main page background. - **Surface containers:** e.g. `#e2efff`, `#edf4ff`, `#ffffff` for bands and cards. - **On-surface:** `#001d32` — primary text. - **On-surface variant:** `#3c4948` — secondary text. - **Outline:** `#6c7a78` — borders, muted labels. - **Error:** `#ba1a1a` (and error container tokens as defined). ## Typography - **Primary stack:** Inter (via `--font-inter` in layout/theme). - **Headlines:** Heavy weights, tight tracking where used on marketing sections. - **Body:** Readable line length; Material-onboarding-style labels for small caps / eyebrow text where applicable. ## Iconography / assets - **Material Symbols** (outlined) where referenced in components. - **Brand marks (ecosystems band):** [`simple-icons`](https://simpleicons.org/) (CC0) for SAP, Google Ads, Google Analytics, Meta; **Devicon** (MIT) for AWS and Azure SVGs (Simple Icons no longer ships those brands in current releases). Rendered **monochrome** using `text-primary` / `fill-current`. ## Design tokens (illustrative) ```css /* Subset — full set in globals.css :root and @theme inline */ :root { --primary: #006a64; --primary-container: #16b2a8; --surface: #f7f9ff; --on-surface: #001d32; --background: #f7f9ff; } ```

Screen specifications (current build) ### Home - **Hero:** Split layout—headline + supporting copy + **ContactInquiryForm** (`variant="hero"`). - **Ecosystems band:** Title **“Tools & ecosystems we operate in”**; flex/grid of brand logos; hover opacity transition on row. - **Downstream sections:** Value props, stats, services teaser, CTAs—per [`stitch-tech-site/src/app/page.tsx`](stitch-tech-site/src/app/page.tsx). ### Services - Hub and child routes under `/services/...` as in app directory. ### Case Studies - Static/marketing page structure as implemented. ### About - **Hero:** Large headline + circular portrait (external image URL as currently coded—**not** the team grid photos). - **Mission, values grid, timeline** as in page. - **Team:** Three cards; **`next/image`** with `fill` + `aspect-[3/4]`; grayscale until hover; names and titles as specified in PRD overview. ### Contact - Two-column layout: copy + **ContactInquiryForm**; optional WhatsApp in contact details when env is set. ### FAQ - Categories and accordion UI; content driven from `faqCategories` data.

Build guide ## Stack - **Framework:** Next.js **16** (App Router). - **UI:** React **19**, **Tailwind CSS v4** (`@import "tailwindcss"` + `@theme inline`). - **Language:** TypeScript. - **Data / APIs:** Prisma + SQLite (Better SQLite adapter); **Nodemailer** for contact email. - **Brand SVG dependencies:** `simple-icons`, `devicon` (see [`PlatformsWeWorkWithLogos.tsx`](stitch-tech-site/src/components/PlatformsWeWorkWithLogos.tsx)). ## Local development - From `stitch-tech-site/`: `npm install`, `npm run dev`, `npm run build`. ## Content & assets - **Team photos:** [`stitch-tech-site/public/team/`](stitch-tech-site/public/team/) (`ajay-headshot.png`, `suraj-headshot.png`, `raman-headshot.png`); source originals may live under repo [`assets/`](assets/) (optional; not required for production build if `public/team` is present). ## Environment - **`NEXT_PUBLIC_WHATSAPP_NUMBER`:** Enables WhatsApp deep links and floating button (see [`stitch-tech-site/src/lib/whatsapp.ts`](stitch-tech-site/src/lib/whatsapp.ts)).
--- ## Out of scope / notes - This PRD describes the **stitch-tech-site** product. Earlier concepts (dark theme, Space Grotesk/Outfit, neon cyan/purple) are **superseded** by the light teal/blue system above unless a separate “concept exploration” doc is created. - Trademark: ecosystem logos are **display-only**; they do not imply official partnership unless separately approved and copy is updated.
